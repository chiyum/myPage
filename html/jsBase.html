JavaScript淺談陳述式及變數

此次介紹最基礎的幾個陳述式及淺談變數。 首先變數的概念是將值賦予在變數上，在以變數去執行各項操作。 像是以上這種概念，將3這個值塞到變數這個盒子內。 此後變數就是代表3。實際操作如下:

var a = 3; b = a + 5; console.log(b);//此時b就等於5。(3 + 5 =8

這項操作被稱為宣告變數在JavaScript是不可或缺的存在，基本的操作都是圍繞著變數 例如. var button = document.querySelector('.nav'); //以上這段程式碼是擷取網頁中class名稱為nav的標籤 button.textContent ='安安你好'; //button是剛才我宣告的標籤名稱，而textContent則是修改文字的代碼。 變數其中一個存在的點就是為了節省每一次要指定元素都得打一大段代碼，省略了這個麻煩。 ---------- Function函式: 函式是甚麼?用處又是? 以生活來講解的話，函式好比是一個任務。 今天假設我是服務生。當客人走入餐廳，正常狀態下都是先招呼客人對吧? 這個招呼的動作就是老闆給服務生的任務，而function函式就是一個空的任務，由開發者來給予。 例如:

function sayWelcome(){ say.歡迎光臨 } 看到客人了 sayWelcome()//執行任務

大概是上面這個概念。假設今天你是老闆，所以你賦予服務生這個任務，當看到客人的時候， 就喊歡迎光臨。 解釋一下上面的代碼: function 這就類似變數的宣告，只是這次不再是盒子，而是任務。 sayWelcome() 則是這個任務的名稱。 {}括號內就是這個任務要執行的內容。 sayWelcome()就是執行任務 當然這個並不是真正的代碼，看到客人這個動作也得寫個代碼才能執行，以上只是解釋概念。 EX.另外function還有個特性就是當執行以後才會把代碼給呼叫出來，執行完成後並刪除。 這個目的主要是為了節省占用的記憶體。 ---------- if判斷式 型態: if(滿足條件){執行內容} 說明: if 其實不難理解，只要把話說白了就很容易理解。 從上述的型態來說()內是條件，只要()內是true也就是滿足條件內容， 那麼{ }內的程式碼就會執行。 例如 1 + 1 == 2 1 + 1 很明顯等於2 2 是否等於 2 答案也是正確也就是true。 那麼if就會執行。 else if(){ }: 那假設有多個if要判斷呢?這個時候else if就派上用場了。 它的用法與if相同，差別是會先判斷完if後再接著判斷else if 若if 不滿足那就接著判斷else if()內的內容，並執行它{}的程式碼 else{ }: 最後呢，當if及else if的條件都不滿足且不執行時，使用else就一定會執行。 (若前面有if有任何一個執行，那else也就不會執行) 假設飽足感分為10分，越高越飽: if(飽足感 <= 3){ 吃便當 //很餓所以吃一顆便當才會飽 }else if(飽足感 > 3 && 飽足感 < 7){ 吃麵包 //只有一點餓，所以吃麵包充飢 }else{ 都不吃 //因為不餓，所以都不吃 } 從上面的範例可以知道，當前面條件都沒滿足，也就是飽足感在7(含7)以上的時候。 就執行都就一定會執行不吃這個動作，這就是else的用途。 EX.for包if，在if{}內只執行滿足if條件的代碼 ---------- switch 型態: let a ='條件' //會套入()內的，這個就是條件 switch(條件){ case '滿足條件': 執行內容 breack; default: 執行的內容 breack } 說明: switch與if有些類似，只要( )內條件與case '滿足條件'內的內容完全相同 那就會執行case與breack之間的程式碼 這個case可以有很多，全由開發者來定，只要()內有對到case ''都會執行。 case與breack: case及breack可以看作是一個人， case就是頭，breack就是腳，缺一不可，而執行的程式碼就是身體。 case就是對條件用，其底下則是執行的程式碼， 而breack則是有中斷case內的程式碼的用途，類似return， 只要是breack後的程式碼都不會執行，白話一點就是這段case的程式碼只執行到這。 default: if有else，switch也有，default就是switch的else， 只要上述條件都沒對上，那麼default就一定會執行，若有其中一個有對上， 那麼則不會運行。 ---------- for迴圈 型態: for(var i = 0;i<次數:i++){ 執行內容 } for介紹可以點此觀看這邊就不再贅述。
