語法作用域 Lexical Scope

JavaScript在作用域中屬於語法作用域，也稱為靜態作用域。 所謂語法作用域就是指，在語法解析時就已經確定了執行範圍。 白話一點就是:在你一開始寫好程式碼的時候就已經，決定了程式碼執行的範圍。 讓我們從以下範例中了解:

var cat = '小貓2號' function sayHI(){ var cat = '小貓1號' console.log(cat) } sayHI() //顯示小貓1號

從以上範例可以得知，若全域變數及區域變數同時都擁有相同的變數下， function會選擇本身擁有的變數下去執行，function這個區域變數本身就是一個作用域。 但是呢!若子層的作用域沒有找到相應的變數時，就會往外尋找相應之變數:

var cat = '小貓2號' function sayHI(){ console.log(cat) } sayHI() //顯示小貓2號

如上顯示，當子層作用域沒有，但父層作用域有變數時，子層作用域就會向外尋找相應的變數， 不過卻無法反過來，也就是說父層無法讀取到子層作用域的變數。

function sayHI(){ var cat = '小貓1號' } console.log(cat) //顯示 is not defined 因為偵測不到這個變數存在

從上述幾點我們只是理解作用域能從子層向外尋找變數，還是不太能清晰的了解作用域的執行範圍， 所以我們繼續看以下的範例:

var cat = '小貓1號' function sayHI(){ console.log(cat) //顯示小貓1號 } function sayHI2(){ var cat = '小貓2號' sayHI() } sayHI2() //還是顯示小貓1號

明明已經將sayHI()置入了sayHI2的function內了，為何還是會顯示小貓1號呢? 因為當撰寫好的syaHI()這個function時就確定了他的執行範圍， sayHI()及sayHI2()就是不同的作用域，若本身沒有變數只會向父層尋找， 並不會因為syaHI()在其他的function內就從而改變。 EX: 這邊補充一點，當我們在子層作用域取得父層作用域的變數並改動變數時， 父層作用域的變數也會因此而改動，這稱為內向作用。

var a = 1 function changeA(){ a= 2 } changeA() console.log(a) //顯示2
