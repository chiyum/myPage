undefined、is not defined、 NaN 差異

當撰寫JS的時候時常會出現以下三種錯誤訊息， 三種錯誤訊息又有甚麼不同呢?差別在哪裡? 說起這個就要先說明提升，這邊簡單的說明一下提升為何。 當JS在運行的時候會分為兩個步驟創造環境與執行環境。 拿以下範例來說明: 變數:

var a = 1 創造階段↓ var a //一開始會先宣告變數 執行階段↓ a = 1 //直到執行階段時才會賦予值

函式:

fu()//依舊會顯示console.log的內容 function fu(){ console.log('測試') } 創造階段↓ function fu(){ console.log('測試') }//在開始的時候就已經準備好執行 執行階段↓ fu()//直到執行階段時立刻觸發

簡單總結一下: 變數是一開始會先建立，但還不會賦予值，直到執行的時候才會賦予。 而函式則是在一開始的時候就先建立好函式，執行階段的時候便直接執行。 所以不論函式的執行位置是在函式陳述式的前或後，它都是可以執行的，因為在創造環境時 函式就已經建構完成，只待執行。 undefined: 提升介紹完了，那麼接下來進入重點，首先先談undefiend。 從提升的介紹我們可以得知，變數在執行的時候才會賦予值。 所以當變數還沒賦予值之前，它就是undefined。

console.log(a)//此時為undefined var a = 1 console.log(a)//此時則為1

為何在變數宣告前會等於undefined呢? 因為在前面有說過，在載入JS的時候，就分為創造與執行兩個階段。 所以載入時已經存在了a這個變數，是等到輪到a = 1這個階段的時候， 才會將1賦予到a上。 因此在宣告後的(也就是執行階段完成後)使用console.log查詢，才能查詢到1這個值。 is not defined: 東西 is not defined 翻譯過來就是找不到這個東西。 也就是說，系統並沒有偵測的東西這個變數。 從上面的例子我們可以看到，當有變數但沒有值的時候會顯示undefined， 而is not defined 就是完全沒有這個變數。 當沒有宣告變數，自然也就沒有這個變數。 NaN: NaN這部分就比較簡單明瞭一些。 NaN = not a number //不是一個數字 由此可知，當出現NaN時，也就代表本該是數字的值卻不是數字。 例如計算的時候: 1 + '我' = NaN
